Automating Curbside Pickup with Deep Learning

Using Unity, Keras, and Google Sheets API to intelligently match cars to online orders in real-time.

Project Overview

Making curbside pickup seamless for everyone involved. Curbside pickup is meant to be easy, but both customers and employees still face friction: customers must notify stores via clunky apps or calls, while staff rush outside to identify cars.


Our goal was to eliminate that friction by making the process intelligent, hands-off, and human-centered. We built CurbsideClassifier to solve inefficiencies in contactless order pickups. Using simulation and AI, we automatically identify arriving vehicles and notify store employees—no phone calls, no apps, no delays.

Design & Development

To train our model, we generated annotated data from Unity. We exported screenshots and JSON metadata of vehicle positions. We trained a Keras neural network to identify vehicles in individual parking spots and detect empty spaces.


After classification, our system used the Google Sheets API to automatically display the matched order and car location inside the store. No human input needed.

Simulation Environment

Using Unity, we modeled an entire parking lot filled with cars, trees, rocks, and other realistic obstructions to simulate real-world conditions and generate training data.

Technical Stack

  • Unity (C#): Simulation and screenshot capture
  • Python + Keras: Deep learning classification model
  • PIL + JSON: Annotation and preprocessing
  • Google Sheets API: Internal display and status sync

Results

Our neural network identified which order was in which spot with 100% accuracy on over 3000 simulated samples. Even in challenging conditions—cars parked across lines or on medians—the model maintained 97% accuracy. These promising results show that with minimal tweaks, CurbsideClassifier could be ready for real-world deployment.

Challenges & Lessons

  • Learning Unity + C# scripting from scratch
  • Custom logic for image slicing and vehicle detection
  • Training a deep learning model on simulated images
  • Integrating with real-time Google Sheets API display

Known Limitations

Our model has not yet been tested in live outdoor conditions. Future iterations will need to account for weather, lighting, car variation, and false positives caused by overlapping shapes.

What We Learned & Future

We aim to collect real images from actual parking lots under various lighting/weather conditions. We also plan to incorporate license plate detection to resolve duplicate car issues and improve robustness in edge cases.

This system has the potential to be used in real businesses—making the pickup process smoother for customers and less burdensome for staff.
